# T-129: Handle edge cases and validations

**Epic:** EPIC-023

## Description
Validate backup operations and handle error scenarios gracefully.

## Acceptance Criteria
- Validate backup path exists and is writable
- Handle missing .m2k folder scenario
- Prevent sync to invalid/restricted paths
- Show meaningful error messages for failures
- Handle disk space issues

## Technical Notes
- Check backup path permissions before sync
- Verify .m2k exists in project before sync
- Prevent circular backup (backup path inside project)
- Catch and report IO errors
- Validate disk space availability

## Dependencies
- T-128

## Testing
- Sync without .m2k folder → appropriate error
- Sync to read-only path → permission error shown
- Sync with insufficient disk space → error handled
- Invalid backup path → validation prevents sync
