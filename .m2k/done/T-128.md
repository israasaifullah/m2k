# T-128: Add dynamic throttle calculation (16ms-100ms)

**Epic:** EPIC-024

## Description
Implement adaptive emit_delay logic that adjusts based on bytes/sec from sliding window. Fast path (16ms) for normal commands, gradual scaling to 100ms for heavy streams.

## Acceptance Criteria
- Calculate emit_delay based on bytes_per_sec:
  - <50KB/s: 16ms (fast path)
  - 50-200KB/s: 33ms (medium)
  - >200KB/s: 100ms (heavy throttle)
- Replace fixed 50ms delay with adaptive delay
- Emit when `last_emit.elapsed() >= emit_delay` OR `batch.len() > 32KB`

## Technical Notes
**File:** `src-tauri/src/pty.rs` (around line 95-100)

**Approach:**
```rust
let emit_delay = if bytes_per_sec < 50_000 {
    Duration::from_millis(16)
} else if bytes_per_sec < 200_000 {
    Duration::from_millis(33)
} else {
    Duration::from_millis(100)
};

if last_emit.elapsed() >= emit_delay || batch.len() > 32768 {
    // emit
}
```
