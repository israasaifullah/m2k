# T-129: Test adaptive throttling across workloads

**Epic:** EPIC-024

## Description
Comprehensive testing of adaptive throttling implementation against various workload scenarios. Verify no regression in simple commands and no crashes in heavy streams.

## Acceptance Criteria
All test cases pass:
1. **Light load:** `ls`, `pwd`, `echo` - <50ms response
2. **Medium load:** `npm install`, `git log --all` - smooth scrolling
3. **Heavy load:** `find /`, `yes | head -1000000` - no crash, controlled throttle
4. **Interactive:** `vim`, `top` - no input lag
5. **Mixed:** Multiple PTYs active - no cross-talk

## Technical Notes
**Success Metrics:**
- Simple commands: no perceptible delay vs baseline
- Heavy streams: no UI freeze
- CPU usage: <10% during massive output
- Memory: stable (no leak)

**Test Commands:**
```bash
# Light
ls -la
echo "test"
pwd

# Medium
npm install
git log --all --oneline

# Heavy
find / 2>/dev/null
cat /dev/urandom | head -c 10M

# Interactive
vim test.txt
top
```
